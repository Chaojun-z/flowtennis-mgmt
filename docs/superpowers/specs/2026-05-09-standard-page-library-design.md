# 后台标准页组件库设计方案

## 1. 目标

本次不是只改某一个页面，而是建立一套后台标准页组件库，后续所有列表页、详情页、编辑页都必须基于这套标准库生成。

核心目标：

1. 新页面不再允许每页自己画一套。
2. 已有页面逐步迁移到同一套标准页体系。
3. 后续如果出现新样式，不允许页面私写，必须先进入标准库，再复用。
4. 允许少量页面级定制，但定制只能挂在标准组件预留的扩展口上，不能重写壳子。

## 2. 范围

第一阶段先建立标准页体系，并选用以下两个页面作为试点：

1. 学员管理
2. 校区管理

本阶段范围仅针对管理端后台页面，不直接覆盖教练网页端。

后续按批次迁移：

1. 简单列表页：教练管理、账号管理等
2. 中等复杂页：班次管理、排课管理、价格管理、购买记录等
3. 复杂页：财务、会员、约球、工作台等

教练网页端边界：

1. 教练网页端不直接套用本次“管理端标准列表页”
2. 教练网页端后续如需统一，应单独定义“教练端页面标准”
3. 教练网页端可以复用底层基础能力，例如：
   - 空值规则
   - toast 反馈
   - 按钮样式
   - 弹窗壳
   - 某些标准单元格样式

## 3. 总体原则

### 3.1 页面生成原则

以后页面开发方式从“写页面”改成“注册页面配置”。

页面文件只负责声明：

1. 页面标题和解释条
2. 统计卡
3. 搜索和筛选项
4. 主操作按钮
5. 表格列定义
6. 分页定义
7. 弹窗定义
8. 少量扩展渲染函数

页面文件不再负责手写整页 HTML 骨架。

### 3.2 统一与扩展原则

采用“80% 强制统一 + 20% 受控扩展”：

1. 页面骨架、表格、分页、弹窗、按钮、反馈交互统一。
2. 页面允许扩展内容，但不能推翻组件壳子。
3. 某种页面私有扩展如果重复出现 2 次以上，必须升级为标准能力。

### 3.3 迁移原则

1. 先搭建标准库，再逐页迁移。
2. 每次只迁 1 个页面，降低爆炸半径。
3. 标准库接入后，老页面逻辑不同时大改。
4. 公共组件稳定后再扩大迁移范围。

## 4. 全局硬规则

以下规则不允许页面自行覆盖：

### 4.1 空值规则

1. 所有无数据场景统一显示 `-`
2. 禁止出现 `—`
3. 适用范围包括：
   - 表格单元格
   - 详情弹窗只读字段
   - 编辑页回显为空的只读信息
   - 统计卡副信息
   - 关联信息区
   - 列筛选回显

### 4.2 筛选口径规则

1. 默认分页展示多少条，与筛选口径无关。
2. 搜索和筛选永远基于当前条件下的全部数据集执行，而不是只在当前页的 20 条数据内筛。
3. 分页只是展示切片，不参与筛选逻辑。
4. 排序也必须基于筛选后的完整结果集执行，再切分页。

标准顺序固定为：

`原始数据 -> 搜索/筛选 -> 排序 -> 分页切片 -> 当前页展示`

### 4.3 分页规则

1. 每页条数标准只允许 `20 / 50 / 100`
2. 切换每页条数后回第一页
3. 搜索、筛选、排序变更后回第一页
4. 删除导致当前页超范围时，自动回退到有效页码

### 4.4 危险操作规则

1. 删除必须二次确认
2. 作废必须填写原因
3. 保存按钮必须防重复提交
4. 保存失败后保留输入内容，不自动关闭弹窗

## 5. 第一版标准库模块

第一版标准库包含以下模块：

1. `PageShell`
2. `ExplainBanner`
3. `StatCards`
4. `FilterBar`
5. `TableToolbar`
6. `DataTable`
7. `Pagination`
8. `StandardModal`
9. `ConfirmDialog`
10. `ToastFeedback`

## 6. 模块设计

### 6.1 PageShell

负责整页骨架，固定区块顺序：

1. 页面标题区
2. 解释条
3. 统计区
4. 搜索/筛选区
5. 主操作区
6. 表格区
7. 分页区

约束：

1. 页面不能自己重排这些区块顺序
2. 列表页不能跳过表格区自建主体区域
3. 分页区不能挪到其它位置

扩展口：

1. `headerExtra`
2. `tableTopExtra`

### 6.2 ExplainBanner

负责页面解释条。

标准能力：

1. 说明文案
2. 轻提示标签
3. 警示型说明
4. 可选链接型说明

用途：

1. 告知当前页面业务口径
2. 告知操作限制
3. 告知统计说明

### 6.3 StatCards

负责统计卡统一展示。

标准字段：

1. `label`
2. `value`
3. `unit`
4. `subText`
5. `status`

统一规则：

1. 数值字号统一
2. 单位位置统一
3. 副说明位置统一
4. 空值仍显示 `-`

扩展能力：

1. 警示卡
2. 成功卡
3. 趋势提示

### 6.4 FilterBar

负责搜索和筛选区域。

支持控件：

1. 搜索框
2. 单选下拉
3. 多选下拉
4. 日期选择器
5. 日期区间选择器
6. 更多筛选折叠区
7. 重置按钮
8. 查询按钮

交互标准：

1. 搜索框固定在筛选区左侧
2. 支持回车触发查询
3. 默认采用“点查询执行”
4. 重置后清空条件并回第一页
5. 查询执行后基于全量结果集筛选
6. 筛选项高度、边框、hover、focus 样式统一
7. 日期选择器和下拉框使用统一视觉与交互
8. 时间选择器、日期选择器、下拉框的打开方向、面板阴影、选中态、禁用态统一
9. 下拉和时间控件清空后的空态文案统一显示 `-`
10. 更多筛选展开和收起的动效、箭头方向、收起后回显规则统一

扩展口：

1. `extraFilters`
2. `filterOptionLoader`

### 6.5 TableToolbar

负责表格上方主操作区。

标准能力：

1. 主按钮，例如新增
2. 次按钮，例如导出
3. 批量操作入口
4. 右侧辅助操作

统一规则：

1. 主按钮位置统一
2. 导出、批量操作等次按钮风格统一
3. hover、disabled、loading 状态统一

### 6.6 DataTable

负责列表页表格主体。

标准能力：

1. 表头
2. 列宽配置
3. 左固定列
4. 右固定操作列
5. 表头固定
6. 横向滚动
7. 空状态
8. 加载状态
9. 主副文本单元格
10. 标签型单元格
11. 金额型单元格
12. 日期型单元格
13. 操作列
14. hover 样式
15. 选中样式
16. 批量选择样式
17. 固定列阴影和分隔边界

表格列标准字段：

1. `key`
2. `title`
3. `width`
4. `align`
5. `fixed`
6. `type`
7. `sortable`
8. `filterable`
9. `render`

#### 6.6.1 列排序标准

标准能力：

1. 支持升序
2. 支持降序
3. 支持取消排序
4. 排序图标统一
5. 当前排序列高亮统一

点击顺序固定：

`默认 -> 升序 -> 降序 -> 默认`

#### 6.6.2 列筛选标准

标准能力：

1. 某列挂载筛选入口
2. 筛选弹层样式统一
3. 已筛选高亮统一

#### 6.6.3 额度/余额单元格标准

将学员管理中的“课包/课时”额度展示、订场用户中的“当前余额”展示收编成标准单元格类型。

新增标准单元格类型：

1. `quota-bar`
2. `amount-bar`

统一规则：

1. 有额度时显示统一样式的浅底进度条
2. 文本展示统一，例如 `16/20` 或 `¥4,826`
3. 无数据仍显示 `-`
4. 0 值与空值区分：
   - 真实 0 显示 `0/总额` 或 `¥0`
   - 无数据显示 `-`
5. 这个样式不是学员页私有能力，而是标准库能力，后续订场用户、会员额度等页面复用

#### 6.6.4 行 hover 与操作显隐标准

标准规则：

1. 行 hover 背景色统一
2. 行 hover 时右侧操作列的强调态统一
3. 如果某类列表采用“hover 后显示更多操作”，则显隐方式、动效、点击热区统一
4. 危险操作在 hover 态下也必须保持危险色和二次确认规则
5. 不允许某页 hover 只高亮文字、某页 hover 整行飘动，交互必须统一

### 6.7 Pagination

负责分页器。

标准能力：

1. 总条数
2. 每页条数切换 `20 / 50 / 100`
3. 页码列表
4. 上一页/下一页
5. 当前页高亮

统一规则：

1. 分页器布局统一
2. 当前页按钮样式统一
3. 切换页码不改变筛选条件
4. 切换每页条数后重新从第一页展示

### 6.8 StandardModal

负责详情、新增、编辑弹窗。

标准模式：

1. `detail`
2. `create`
3. `edit`

标准能力：

1. 标题
2. 宽度档位：窄 / 中 / 宽
3. 表单两列布局
4. 区块标题
5. 只读文本块
6. 必填标识
7. 校验提示
8. 底部按钮区
9. 保存中状态
10. 关闭前未保存提醒
11. Esc / 点击遮罩关闭规则统一
12. 表单 focus 顺序和错误滚动定位统一

按钮规则：

1. 左侧可放危险操作按钮
2. 右侧放取消和主按钮
3. 主按钮固定在最右

区块组织方式：

1. 基本信息区
2. 业务信息区
3. 关联信息区
4. 备注区
5. 自定义扩展区

### 6.9 ConfirmDialog

负责确认型弹窗。

标准场景：

1. 删除确认
2. 作废确认
3. 影响说明确认

标准能力：

1. 标题
2. 风险文案
3. 影响数量提示
4. 原因输入框
5. 确认按钮 loading

### 6.10 ToastFeedback

负责全局反馈。

标准类型：

1. 成功
2. 失败
3. 警告
4. 信息

统一规则：

1. 文案风格统一
2. 展示位置统一
3. 时长统一

4. 默认值
5. 查询行为

### 7.4 actions

1. 新增按钮
2. 导出按钮
3. 批量操作按钮
4. 额外操作按钮

### 7.5 table

1. 列定义
2. 排序定义
3. 列筛选定义
4. 行操作定义

### 7.6 pagination

1. 默认每页条数

### 7.7 modals

1. 详情弹窗
2. 新增弹窗

### 7.8 data

## 8. 交互流程标准

### 8.1 列表渲染流程

统一流程固定为：

`原始数据 -> 搜索/筛选 -> 排序 -> 分页 -> 当前页渲染`

### 8.2 搜索流程

1. 输入关键字
2. 按回车或点查询
3. 基于全量结果集过滤
4. 回第一页
5. 重新渲染统计、表格、分页

### 8.3 筛选流程

1. 修改筛选条件
2. 点查询或按统一规则自动执行
3. 基于全量结果集筛选
4. 回第一页
5. 重新排序、分页、渲染

### 8.4 保存流程

1. 点击保存
2. 按钮进入 loading
3. 禁止重复点击
4. 保存成功后关闭弹窗
5. 刷新当前页面对应数据
6. 重新计算统计、筛选结果和分页

### 8.5 删除流程

1. 点击删除
2. 二次确认
3. 如需原因则输入原因
4. 删除成功后刷新列表
5. 如当前页空掉则自动回到有效页码

## 9. 首批试点页面要求

### 9.1 学员管理

学员管理必须验证以下能力是否完整：

1. 解释条
2. 统计卡
3. 搜索
4. 多下拉筛选
5. 标准表格
6. 额度条单元格
7. 右侧固定操作列
8. 详情弹窗
9. 编辑弹窗
10. 保存和删除交互

### 9.2 校区管理

校区管理用于验证轻量标准页是否足够简洁：

1. 少量筛选或无复杂筛选
2. 简单表格
3. 标准新增/编辑弹窗
4. 标准分页

## 10. 非目标

本阶段不做以下事情：

1. 一次性重构所有页面
2. 改动现有核心业务规则
3. 改动数据库结构
4. 为了组件化而重写所有页面逻辑

## 11. 风险与控制

主要风险：

1. 老页面结构和新页面结构并存一段时间
2. 已有测试依赖 HTML 结构，迁移时需要同步调整
3. 公共组件改动可能影响已接入页面

控制方式：

1. 先做标准库，不一次性替换全站
2. 每次只迁一个页面
3. 试点页稳定后再扩大范围
4. 公共组件改动必须配套验证已接入页面

## 12. 第一版实施顺序

1. 先建立标准库基础模块骨架
2. 先接入学员管理
3. 再接入校区管理
4. 验证稳定后抽象迁移规范
5. 再迁移其它列表页

## 13. 最终结论

这次要建立的不是“几个公共样式”，而是一套后台标准页生产方式。

以后新增页面应遵循以下规则：

1. 优先用标准库拼页面
2. 标准库不够时，先补标准库
3. 页面不得直接绕开标准库自行画整页

只有这样，页面结构、交互、表格、弹窗、分页、额度条样式才能长期稳定统一。
